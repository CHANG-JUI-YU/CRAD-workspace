# self-character-card skill — 設計規格

## 專案概要

| 項目 | 設定 |
|------|------|
| Skill 名稱 | `self-character-card` |
| 輸出產物 | 角色卡（JSON / PNG） |
| 目標用戶 | 自用 |
| 與 tavern-cards 的關係 | 互補並存，創作層由本 skill 負責，技術打包層分流至 tavern-cards skill |

## 定位

`self-character-card` 與現有 `tavern-cards` skill 互補並存，不重疊不衝突：

- **創作層（本 skill）**：負責角色模塊創作 — 外顯、內質、外延、外延擴展、特質細化、場景語料、自我介紹
- **技術層（tavern-cards skill）**：負責 forge CLI 操作、世界書條目配置、打包流程、MVU/EJS 等
- 日常創作流程：本 skill 決定「寫什麼」→ 產生 YAML 設定 → 分流至 tavern-cards skill 決定「怎麼包」

## 創作流程

完整的創作流程分為四個階段：

### 階段 A：需求對齊

1. 確定專案屬性（專案名稱、單角色/多角色）
2. 列出角色清單，每個角色標註需要創作的模塊（可選全 7 或部分）
3. 產出 `創作規劃.yaml`，寫入專案目錄

### 階段 B：模塊創作循環

對每個角色的每個選定模塊，依序執行：

1. 從 SKILL.md 讀取該模塊的精簡骨架（內嵌 YAML 模板，僅有字段標題）
2. 如需要完整 COT 思維鏈、參考理論列表、質量檢查項目，按需讀取 `references/module-xxx.md`
3. 在該角色目錄下產出設定檔，例如 `外顯.yaml`、`內質.yaml`
4. 執行該模塊自帶的質量檢查

每個模塊的統一創作步驟：
- **COT 思維**：分析需求 → 識別約束 → 確定參考作品（4 個）→ 確定參考理論（3 個領域）→ 分析角色 → 整體檢驗
- **輸出格式**：中文 YAML
- **質量檢查**：模板完整性、前後一致性、原型還原度、多角度信息完整度

### 階段 C：整合檢驗

- 同一角色跨模塊檢查（外顯 vs 內質 vs 外延 不矛盾）
- 多角色之間的關係一致性
- 輸出檢驗報告

### 階段 D：Forge 打包

1. 建立 forge 專案目錄結構（`init`）
2. 將各角色 YAML 設定文件轉為 forge 可讀的世界書條目
3. 執行 `forge configure` 推導運行時字段
4. 執行 `forge pack` 產出角色卡 JSON/PNG
5. 具體操作步驟參照 `tavern-cards` skill 的 forge CLI 指引

## 參考文件結構

每個 `references/module-xxx.md` 文件統一格式：

1. **COT 思維鏈**（英文輸出，含分析需求、識別約束、優先級排序）
2. **核心概念辨析**（該模塊的定義與邊界）
3. **4 個參考作品指引**（商業/藝術/非主流作品，含選擇原因）
4. **3 個參考理論領域**（符合角色風格的理論基礎）
5. **質量檢查清單**（模板完整性、一致性、原型還原度、多角度評估）

## 多角色創作規劃

創作規劃.yaml 的結構：

```yaml
專案名稱: ''
輸出格式: charactercard
角色列表:
  - 名稱: ''
    模塊: [外顯, 內質, 外延, 外延擴展, 特質細化, 場景語料, 自我介紹]
forge 專案目錄: ''
```

創作順序：先確定角色清單 → 依序對每個角色按 外顯→內質→外延→外延擴展→特質細化→場景語料→自我介紹 的順序創作 → 全角色完成後跨角色檢驗 → forge 打包。

## 與 tavern-cards skill 的銜接點

| 時機 | 動作 | 參照 |
|------|------|------|
| 打包前 | 建立 forge 專案目錄 | tavern-cards: forge init |
| YAML → 條目 | 將設定文件轉為世界書條目 | tavern-cards: 條目創作流程 |
| 打包 | 產出角色卡 JSON/PNG | tavern-cards: forge pack |
| 配置 | 推導運行時字段 | tavern-cards: forge configure |
