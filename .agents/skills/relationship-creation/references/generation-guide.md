# Relationship Generation Guide

1. 以 Blueprint `relationships.character_ids` 的順序建立 exact participant set，至少兩人。
2. 保留 engine 提供的六碼 `team_code`；revision 不得重算或變更。
3. 每位 participant 恰有一份 summary；每個 source-target 有序 pair 恰有一格，包含 self-pair。
4. 相反方向分別依各自認知、動機、權力與盲點撰寫，不做對稱複製。
5. 群組可重疊；每組寫清形成原因、運作方式、排他性、潛在衝突與加入條件，成員不可越界。
6. network summary 明確描述群組間關係、穩定性、具體衝突觸發與嚴重度、親密發展機會。
7. 只使用 Blueprint、accepted facts 與 participant exact modules；創作補完不可偽裝成來源事實。
8. 提交 structured YAML value；不輸出 thinking、roleplay、場景或 compiler `<team_CODE>` 包裝。
