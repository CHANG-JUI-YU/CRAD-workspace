# 模組 4：特質細化 `trait_refinement`

## 核心責任

把抽象性格標籤拆成可觸發、可表演、可跨情境辨識的行為機制，防止角色退化成單一標籤。

## `data` 固定格式

```yaml
性格特質:
  - 特質群名稱: ''
    群描述: ''
    包含特質: []
```

## 特質結構

建立 2 至 4 個特質群，例如核心驅力、防禦機制、社交面具、親密模式。所有群組合計必須 5 至 8 個特質。

每個特質包含：

- 特質名稱。
- 來源：它由哪些經歷、需求、價值或自我敘事形成。
- 心理深度：極深／深／中／淺。
- 觸發方式：人物、語句、環境、損失、權力或親密條件。
- 描述：與其他特質如何合作或衝突。
- 外在表現：至少 3 個可觀察行為、語句或身體訊號。

## 建議格式

```text
特質群：防禦機制
群描述：她透過先發制人的評價維持對局面的掌控。

特質：挑釁式試探
來源：害怕在投入後才發現自己被輕視。
心理深度：深
觸發：對方回應含糊、延遲承諾或轉移視線。
描述：逼迫對方立刻表態以減少不確定性，也會壓制她想直接求證的衝動。
外在表現：故意誇張質疑；縮短句子；靠近後突然拉開距離。
```

## 品質檢查

- 不把同義詞拆成多個假特質。
- 每個特質都有觸發與外在表現。
- 特質之間存在矛盾、轉化或層級，不是平鋪清單。
