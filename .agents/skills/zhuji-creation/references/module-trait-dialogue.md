# 模組 5：特質語料 `trait_dialogue`

## 核心責任

把模組 4 的 5 至 8 個核心特質轉成可直接模仿的長篇角色語料，固定角色的節奏、詞彙、內在自我認知與各特質造成的結果。這不是場景語料，不依特定地點、事件或關係階段分類。

## `data` 固定格式

```yaml
人物說話節奏: ''
人物語言習慣:
  自稱: ''
  口頭禪: ''
  特殊詞彙偏好: ''
  方言痕跡: ''
  語氣助詞使用: ''
  語言情感程度: ''
  用詞程度選擇: ''
扮演關鍵要點: []
Traits:
  - Trait_Name: ''
    Embodiments: []
    inner_thought_prompt: ''
    instant: []
    Results: []
```

## 硬性規則

- `Traits` 必須 5 至 8 項，對應模組 4 的核心特質；固定用 `Trait_Name`，不得改成動態 YAML key。
- 每個 Trait 的 `instant` 必須 3 至 5 條；每條至少 30 個 Unicode 字元，長度依自然語氣決定，不為湊字數灌水。
- 每條語料使用符合角色節奏的逗號、句號、問號、驚嘆號、省略號或換行自然斷句；任兩個斷句標點間不得連續超過 60 個字元。
- `instant` 可為直接交流、宣言、連續發言或直接內心獨白。
- `instant` 禁止動作、神態、場景、旁白、心理狀態解說及第三人稱描述。
- `Embodiments` 寫特質定義、來源與人格位置，不是語料。
- `inner_thought_prompt` 是一句代表核心的內在自我認知，可省略。
- `Results` 寫該特質對內心、行為、能力與社會關係造成的結果，不受語料最低長度限制。

## 品質檢查

- 每種特質的句長、攻防方式、情緒暴露及用詞有可辨識差異，但仍屬同一角色聲線。
- 不複製模組 4 的分析文字；語料要真正展示特質。
- 不提前承擔模組 6 的場景、情緒、對象與關係階段分類。
