# 模組 1：外顯 `appearance`

## 核心責任

只寫可由立即接觸觀察到的資訊：外貌、體態、聲音、衣著、動作、觸感、氣味與生理反應。不要解釋內在動機、創傷史或關係發展。

## `data` 填寫指引

### `appearance-core` 外顯核心

- 姓名、性別、種族／民族、可見身分或職業線索。
- 一句整體概括與 3 至 8 個外顯核心標籤。
- 依 Blueprint 與已核准成人基調具體寫出親密傾向；不可用空泛標籤代替可感知特徵。

### `face-and-expression` 面貌與表情

- 眼睛、鼻耳、嘴唇、眉毛、輪廓、髮色、髮型與辨識標記。
- 習慣表情、眼神、微表情、情緒表達幅度與頻率。
- 使用第三方可觀察描述，不在此推導心理原因。

### `body-and-sensory` 身體與感官

- 年齡、身高、體態、膚色、身材比例；數值只在 Blueprint 或 accepted facts 已確認時使用。
- 手、腳、腿、腰腹、臀、肌肉或其他重要部位的外觀、觸感、氣味與裝飾。
- 成人身體細節與敏感區只在設定明確要求時展開，並保持與背景及既有內容一致。

### `voice-and-language` 聲音與語言習慣

- 音色、音調、音量、聲線與情緒變化。
- 語速、斷句、思考停頓、自稱、口頭禪、詞彙與語氣助詞。
- 對話稱呼本土化，不機械重複 `{{user}}`。

### `clothing-and-movement` 衣著與肢體

- 日常風格、典型上下身、鞋履、配件與場合差異。
- 習慣動作、無意識動作、情緒化姿態、肌肉與應激反應。

### `representative-impressions` 整體感官體驗

- 綜合至少三種感官形成角色第一印象。
- 提供 3 至 5 個不同情境下的代表性外顯片段，避免重複同一姿態。

## 品質檢查

- 每個描述都能回答「旁觀者如何看到、聽到、聞到或觸知」。
- 不以精確數值製造虛假權威。
- 不把性格標籤當作外貌描述，不把背景故事塞入外顯。
