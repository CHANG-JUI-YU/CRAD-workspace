# 模組 6：場景語料 `scene_dialogue`

## 核心責任

依模組 5 的特質聲線，建立關係階段、場景、情緒與不同對象下的長篇角色語言或直接內心獨白。

## `data` 固定格式

```yaml
核心標籤與特質的風格表現:
  角色說話節奏: ''
  角色語言習慣: ''
  扮演關鍵要點: []
  標籤: []
對 {{user}}:
  初始關係與態度: []
  初始認知與在意程度: []
  是否想要進一步關係: []
  分階段好感度:
    初始關係: []
    好朋友: []
    戀人未滿: []
    戀人: []
    摯愛: []
場景刻畫: []
情緒表現: []
面對不同對象: []
```

## 硬性規則

- 除 `角色說話節奏`、`角色語言習慣` 外，語料正文只能是角色直接說話或直接內心獨白。
- 禁止星號、括號動作、旁白、第三人稱心理分析、鏡頭、場景說明或替 `{{user}}` 說話。
- 每段應形成完整語氣弧線，而不是十幾句互不相干的短句。
- 每個語料字串至少 30 個 Unicode 字元，長度依場景與情緒自然展開，不為湊字數拼接無關句子。
- 使用符合角色節奏的標點或換行自然斷句；任兩個斷句標點間不得連續超過 60 個字元。

## `data` 填寫指引

### `speech-profile` 語言輪廓

- 說話節奏、斷句、停頓與情緒影響。
- 自稱、口頭禪、詞彙、方言、語氣助詞與稱呼變化。
- 3 至 8 個標籤，每個以 `{ 標籤名稱, 語料[] }` 表達。

### `relationship-dialogue` 對 `{{user}}` 的分階段語料

依 Blueprint 適用的關係路徑建立階段。可使用：

- 初始關係。
- 好朋友／穩定同伴。
- 戀人未滿或關係曖昧。
- 戀人／深度親密。
- 摯愛／長期信任。

只有初始關係為朋友或更疏遠時才生成 `分階段好感度`；每個固定階段恰好 3 條語料。若不適用，省略整個 `分階段好感度`。

### `scenario-dialogue` 情境語料

`場景刻畫` 選最可能發生的 3 個場景，以 `{ 場景, 語料[] }` 表達；`情緒表現` 選最常見的 3 種情緒，以 `{ 情緒, 語料[] }` 表達；`面對不同對象` 選最常面對的 3 類對象，以 `{ 對象, 語料[] }` 表達。

## 本土化

- 描述結構可用 `{{user}}` 定位；實際台詞依親疏使用「你」、職稱、暱稱或角色專屬稱呼。
- 不殘留「あなた」「お前」、羅馬音、「君」「桑」「醬」或「{{user}}君」等生硬直譯，除非 Blueprint 明確把它設定成角色刻意使用的語言特徵。

## 品質檢查

- 刪掉標題後，每段仍是純角色台詞。
- 不同階段的句長、攻防、稱呼與暴露程度確實不同。
- 台詞能反映模組 2 與模組 5 的心理機制。
