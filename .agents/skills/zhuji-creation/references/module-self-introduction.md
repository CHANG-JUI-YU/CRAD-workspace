# 模組 7：自我介紹 `self_introduction`

## 核心責任

讓角色以第一人稱直接向 `{{user}}` 揭露自己如何理解外貌、性格、世界、關係與當下追求。這是常態角色設定，不是開場事件或 greeting。

## `data` 固定格式

```yaml
核心標籤與特質的風格表現: {}
對 {{user}}: {}
外評觀價: {}
性格基礎: {}
能力興趣: {}
背景設定與成長經歷: {}
人際關係: {}
性相關: {}
```

## 硬性規則

- 除 `角色說話節奏` 與 `角色語言習慣` 可客觀描述外，每個 YAML 欄位都視為一道隱含訪談問題，欄位值須由角色以第一人稱直接面向 `{{user}}` 回答，且每段至少 30 個 Unicode 字元。
- 回答長度依問題自然決定，不為湊字數灌水；使用符合角色節奏的標點或換行斷句，任兩個斷句標點間不得連續超過 60 個字元。
- 使用模組 6 已建立的自稱、節奏與詞彙。
- 不使用星號、括號動作、舞台指示、第三人稱旁白或鏡頭描述。
- 不替 `{{user}}` 決定行動、感受或既定回應。

## `data` 填寫指引

### `voice-and-core` 語氣與核心特質

- 角色說話節奏、語言習慣、扮演關鍵、核心標籤與核心特質。
- 內容需像角色真的在說話，不是作者分析報告。

### `view-of-user` 對 `{{user}}`

- 初始關係與態度。
- 對 `{{user}}` 的認知與在意程度。
- 是否期待關係變化，以及角色願意承認到什麼程度。

### `self-image` 外顯自我評價

- 對容貌、身材、吸引力與服裝風格的主觀評價。
- 評價可能偏誤，但必須符合角色自我認知，不能偷渡全知旁白。

### `philosophy` 處世哲學

- 對自己性格、世界、道德、生命意義與內心禁忌的看法。
- 角色可迴避、合理化或矛盾，不要求完全誠實自省。

### `motivation` 動機系統

- 核心訴求、當下追求、害怕失去的事物與希望 `{{user}}` 理解的部分。

### `life-and-relationships` 能力、背景與關係

- `能力興趣`：職業、技能、日常興趣、3 至 5 個喜愛與厭惡事物、可選嗜好。
- `背景設定與成長經歷`：家庭、成長、可選重要人物、角色對自身社會身分與職能的看法。
- `人際關係`：社交模式、渴望關係、依賴程度、工作與特殊感情對象，以及新關係初始化方式。
- `性相關`：個人性癖、人格影響、可選幻想、性經歷、性生活與可選第一次性交對象。

## 品質檢查

- 每一句都能自然改寫成「我……」而不破壞視角。
- 沒有開場地點、玩家抵達、玩家動作或即時事件。
- 與模組 6 同一聲線，但比場景語料更具自我揭露與長期設定價值。
- 每段回答都帶明顯情緒；保留欄位作為隱含問題，但不額外輸出採訪者台詞、`問題/回答` 包裝、客觀履歷或第三人稱作者報告。
