# Greetings Negative Rules

檢查替玩家宣告思想、感受、對白或既成行動的 Puppeteering；檢查封閉式結局、缺乏可回應切口、角色聲線失真、alternate 只換措辭、group-only 歸屬錯誤。任何把珠璣模組7當作 greeting 的內容都是契約錯誤。

若 greeting 只有簡短場景加一句提問、缺少具體事件或衝突、沒有張力推進，或玩家雖能回答卻沒有足夠局面可互動，應提出附 exact evidence 與補強方向的品質 finding。350 至 600 個中文字與至少三個自然段落是 Creator 的軟目標，不是 normative/schema門檻；不得只按字數判錯，也不得要求同義反覆或空泛灌水。

若使用者與已批准 Blueprint 未明確要求外語，Greeting 敘事、對話或內心語句混入日文原字、假名、羅馬字、日語語尾或其他未翻譯外語，引用 exact evidence 提出語言一致性品質 finding。角色國籍、所在地與姓名風格不是外語授權；正式姓名與必要專有名詞可保留原文。此項不得冒充 normative/schema error。
