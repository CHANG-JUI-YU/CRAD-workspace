# Character Negative Rules

檢查事實矛盾、模式欄位缺漏、跨模組無效重複、只有抽象標籤而無行為證據、語料同質化、關係或動機斷裂，以及超出 Blueprint 的無來源主張。珠璣描述性欄位若大量退化為程度詞、單句結論、模板化形容詞，且缺乏形成機制、觸發、具體表現、情境差異或行為影響，應以有明確 evidence 與補強方向的品質 finding 指出；這不是 normative/schema error，也不得要求姓名、數值、分類等原子事實無意義擴寫。珠璣模組7若包含開場場景、玩家行動或 greeting 結構，列為不可忽略的契約 finding。Normative schema、provenance 與權限問題不可覆寫。

若使用者與已批准 Blueprint 未明確要求外語，新創作正文或語料出現日文原字、假名、羅馬字、日語語尾或其他未翻譯外語，應引用 exact evidence 提出品質 finding。角色國籍、所在地、文化背景與姓名風格不是外語授權；正式姓名、必要專有名詞、schema 鍵與 exact evidence 不列為違規。此項是語言一致性品質問題，不得冒充 normative/schema error。

Blueprint 啟用 relationships 時，缺少 artifact、participant set 或順序不符、角色摘要缺漏、N x N 有序矩陣缺格／重格／缺 self-pair、把 A->B 與 B->A 當成必然對稱、群組含非 participant 或重複成員，均屬不可忽略的契約 finding。另逐格比對角色基礎與各自 required mode modules；群組形成原因、運作模式、排他性、潛在衝突、加入條件及 network summary 若互相矛盾，引用 exact evidence 提出一致性 finding。角色可同時屬於多組，不得把重疊本身判錯。
